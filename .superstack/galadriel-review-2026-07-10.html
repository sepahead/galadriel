Ten-perspective review

#PerspectiveQuestion challengedOutcome
1EstimandAre modalities actually comparable?Native polar/Cartesian innovations are never pooled. Cross-channel work requires an explicit common projection with frame/context/prior provenance.
2NumericsCan NaN, overflow, covariance roundoff, or tail cancellation fabricate evidence?Finite/domain checks, scale-aware covariance validation, stable correlation, `statrs` chi-square tails, and normalized CUSUM units close these paths.
3Temporal stateCan stale, sparse, reordered, or restarted streams look healthy?Strict sequence/timestamp increase, gap resets, expected-modality freshness, bounded tracks, explicit epoch reset, and terminal-value rejection fail closed.
4AttributionDoes a convenient pair or one axis manufacture a culprit?Signed unique strict-majority consensus is required. All runtime axes are assessed; conflicts and positive-plus-insufficient combinations become `Anomaly`.
5AdversaryWhat if the attacker knows the detector?Matched-threshold studies use independent holdout data, while colluding-majority and consistency-preserving blind spots are explicitly retained.
6InferenceDo reported intervals describe the statistic actually shown?Alarm-ranked AUC is consistent across main and CI reports; resample minima are enforced; pointwise scans are not sold as simultaneous proof.
7ResourcesCan valid input trigger unbounded memory or estimator work?Tracks, observations, frames, windows, JSONL bytes/records/lines, live streams, bootstrap work, grids, and PID quadratic fits all have explicit ceilings.
8ConcurrencyCan callback panic/reentrancy/reset sequencing deadlock or corrupt epochs?Panics are contained and counted; direct cross-subscription resets are rejected in callback context; in-flight resets fail fast and can be retried.
9EcosystemDoes any sibling project actually satisfy the claimed seam?No executable downstream consumer imports Galadriel. Crebain is the only producer candidate and currently supports baseline smoke only.
10Security and truthCould the project be interpreted as production-ready despite missing controls?CI and supply-chain policy are hardened; docs now distinguish code correctness, producer truth, transport auth, field validation, and automatic control.

Material defects resolved in this loop

Resolved · data integrity

Mixed-frame and sequential-prior residuals were accepted as cross-channel columns

Added ConsistencyProjection, removed native-innovation fallback, validates every axis and provenance tuple, and detects prior reuse across the entire bounded input—not just the retained tail.

Resolved · attribution

Absolute best-peer correlation could treat sign flips or ambiguous dyads as corroboration

Correlation is signed, family-controlled, and requires one unique strict-majority positive clique. Degeneracy, ties, bridges, and failed peers remain insufficient.

Resolved · statistical inference

PID confirmation could silently become structurally unresolvable

All edges are still fitted, but confirmation uses joint worst-consensus and worst-candidate resampled extrema. Rank resolution is preflighted before estimator work, including multi-axis alpha splitting.

Resolved · evaluation

Latency skipped the capture tail; AUC intervals used a different score; adaptive thresholds reused calibration data

The final frame is always probed once, CIs target the alarm-ranked score, and independently seeded clean holdout trials report observed FAR with Wilson intervals.

Resolved · concurrency

Cross-subscription callback resets could deadlock

A callback-context guard covers every subscription on the current thread, while a reset against active delivery returns a typed retryable error instead of waiting behind a callback that may join it.

Resolved · observability

A clean terminal tail erased earlier replay failures

Replay now retains count/first/last history for alarms, insufficient evidence, missing projection, too few modalities, extraction errors, and analysis errors—even when the terminal verdict recovers.

Resolved · supply chain

CI trust and advisory policy were underspecified

Actions are SHA-pinned with read-only permissions, packages are unpublished, source/license/advisory policy is checked, Dependabot is enabled, and the one disabled-feature exception has a mechanical expiry.

Residual blockers and exact next fixes

Medium · producer integrity

Crebain does not emit the required estimand

Instrument the fusion step before association/gating and before sequential modality updates. Every modality in a frame must be projected from one frozen prior into one registered frame, with a unique shared prior ID.

consistency_projection: Some(ConsistencyProjection {
    values: residual_in_registered_common_frame,
    dimensions: 3,
    frame_id: REGISTERED_FRAME_ID,
    context_id: PROJECTION_SCHEMA_VERSION,
    prior_id: unique_frozen_prior_id,
})
Medium · live security

The Zenoh sidecar is not yet an operationally authenticated path

The consumer now uses NCP's ACL-covered sensor/galadriel-pid route, a strict versioned session/producer envelope, and an explicit secure transport mode. Add the Crebain publisher, deploy mTLS identities, use a fresh session ID for every process epoch, and add heartbeats. Test traffic, denial, forged payload, restart, queued bytes, and all-modal silence end to end.

Medium · supply chain

RUSTSEC-2026-0041 remains transitively present

The affected compression feature is disabled and CI-enforced. Upgrade NCP/Zenoh before 2026-10-01, then remove the exception:

# after a compatible upstream release
ncp-zenoh = { git = "https://github.com/sepahead/NCP", tag = "vNEXT" }
# delete the RUSTSEC-2026-0041 entry from deny.toml

cargo audit
Research gate

No synthetic suite can establish field performance

Preregister a recorded, pre-gate study with explicit misses/rejections, benign maneuvers, producer/transport restarts, independent calibration/holdout sets, and confidence intervals. Keep consistency-preserving and colluding-majority attacks in the red-team library.
